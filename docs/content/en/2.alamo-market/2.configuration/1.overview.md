Everything gameplay-related lives in **`config.lua`**. After any change, restart the resource:

```txt
restart alamo-market
```

## The config at a glance

| Setting                        | Purpose                            | Page                                                          |
| ------------------------------ | ---------------------------------- | ------------------------------------------------------------- |
| `Config.Debug`                 | Verbose console logging            | This page                                                     |
| `Config.Locale`                | UI / notification language         | [Localization](/en/alamo-market/advanced/localization)        |
| `Config.ServerLabel`           | Branding shown in the NUI header   | [Branding](/en/alamo-market/configuration/branding)           |
| `Config.DefaultPaymentMethods` | Fallback payment methods           | [Payments](/en/alamo-market/advanced/payments)                |
| `Config.Marker`                | Global marker appearance           | [Markers](/en/alamo-market/configuration/markers)             |
| `Config.Interaction`           | Distance + open key                | [Interaction](/en/alamo-market/configuration/interaction)     |
| `Config.Notify`                | Notification system                | [Notifications](/en/alamo-market/advanced/notifications)      |
| `Config.ShopCatalogs`          | Reusable item lists                | [Catalogs](/en/alamo-market/configuration/catalogs-and-items) |
| `Config.ShopLocations`         | Where shops are + which catalog    | [Locations](/en/alamo-market/configuration/shop-locations)    |
| `Config.CustomShops`           | One-off shops with their own items | [Custom Shops](/en/alamo-market/advanced/custom-shops)        |

## Debug mode

```lua [config.lua]
Config.Debug = false
```

Set to `true` to print extra diagnostics to the console (notify fallbacks, locale loading, etc.). Leave `false` in production.

## Editing rules of thumb

- Keep the Lua structure valid — commas between table entries, matching `{ }`.
- Strings use quotes: `'cash'`. Names with an apostrophe use double quotes: `"Rob's Liquor"`.
- Coordinates are `vector4(x, y, z, heading)`.
- Restart the resource after every change.

::warning
A single missing comma or bracket will stop the whole file from loading. If shops disappear after an edit, check the server console for a Lua syntax error.
::
