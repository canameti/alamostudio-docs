## 1. Add the resource

Place the `alamo-market` folder inside your server's resources directory, for example:

```txt
resources/[alamo]/alamo-market
```

::note{color="primary"}
The folder must be named `alamo-market`.
::

## 2. Ensure it after your framework

Add the resource to your `server.cfg` **after** your framework core so detection works:

```cfg [server.cfg]
# framework first
ensure qbx_core        # or qb-core / es_extended

# then the market
ensure alamo-market
```

## 3. Pick your language (optional)

Open `config.lua` and set the locale. English is the default:

```lua [config.lua]
Config.Locale = 'en' -- 'en', 'de', 'tr' or any custom file in locales/
```

## 4. Start / restart

Start the server, or hot-restart in your live console:

```txt
restart alamo-market
```

## 5. Verify

You should now see:

- Map **blips** for all 20 shops (LTD, Rob's Liquor, 24/7).
- A **marker** when you walk near a shop.
- The **NUI** opening when you press `E` inside the marker.

::tip{color="primary"}
That's it — the shops are already configured and selling. From here you only customize `config.lua` and `variables.css`.
::

## Quick sanity checklist

- Framework core (`qbx_core` / `qb-core` / `es_extended`) starts before `alamo-market`.
- All catalog item `name`s exist in your inventory.
- No startup errors in the server console mentioning `alamo-market`.

If something does not appear, see [Troubleshooting](/en/alamo-market/deployment/troubleshooting).
